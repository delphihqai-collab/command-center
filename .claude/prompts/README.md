# Prompts

Task-level instructions. Each prompt is a focused, one-off or situational instruction for a specific type of task.

Prompts are thin. They reference skills and rely on instructions for behavioral rules. They do not duplicate either.

## Prompts to Create (post-build)
- `add-page.md` — task prompt for adding a new page to the backoffice
- `add-table.md` — task prompt for adding a new table to the DB schema
- `debug-rls.md` — task prompt for diagnosing a Row-Level Security issue
- `generate-types.md` — task prompt for regenerating Supabase TypeScript types after a migration

## Do Not Create Prompts Until Instructed
